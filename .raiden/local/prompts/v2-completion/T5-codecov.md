## Prompt

You are working in the repository at `/Users/dante/Citadel/BIND`.

BIND is a Python audiobook metadata daemon (v1.7.1). A Codecov account has
been set up, `StarlightDaemon/BIND` is linked, and `CODECOV_TOKEN` is stored
as a GitHub Actions secret.

### Task

#### 1. Add Codecov upload to `.github/workflows/ci.yml`

In the `test` job, immediately after the "Run tests with coverage" step, add:

```yaml
      - name: Upload coverage to Codecov
        uses: codecov/codecov-action@v4
        with:
          token: ${{ secrets.CODECOV_TOKEN }}
          files: ./coverage.xml
          fail_ci_if_error: false
```

`coverage.xml` is already generated by the existing `--cov-report=xml` flag
in the pytest step. No other changes to the pytest step are needed.

#### 2. Add Codecov badge to `README.md`

Locate the badge row near the top of `README.md` (the line(s) with existing
shields.io badges). Add the Codecov badge at the end of that row:

```markdown
[![codecov](https://codecov.io/gh/StarlightDaemon/BIND/branch/main/graph/badge.svg?token=CODECOV_GRAPH_TOKEN)](https://codecov.io/gh/StarlightDaemon/BIND)
```

Replace `CODECOV_GRAPH_TOKEN` with the graph token from the Codecov dashboard
(this is a read-only display token, different from the upload token, safe to
embed in a public README). The operator must supply this value.

### Exit Criteria

- `.github/workflows/ci.yml` `test` job has a `codecov/codecov-action@v4`
  step after the pytest step
- `README.md` has a Codecov badge in the existing badge row
- No Python source files modified
- `ruff check src/ tests/` passes
